## [0.1.0] - 2026-04-29

Initial release — three pillars: generator, /meet, evaluator.

### Added

- **`/team-gen` skill** — generate a complete `.claude/` agent team (orchestrator + specialists, skills, rules, hooks) from a product description. Picks archetypes, fills templates, scaffolds, runs the static linter on every produced file before exit. End-to-end test: a 4-agent Sample Shop team scores A 100/100 across the board. (#4)
- **`/meet` skill** — streamed multi-agent meeting on a topic. Each agent answers in a structured format (constraint check, top-3 risks, dependencies, open questions, effort). Orchestrator consolidates to a global top-10 and writes minutes to `.meetings/YYYY-MM-DD-{topic}.md`. (#7)
- **`/evaluate-agent` skill — static linter (Phase 1)** — citation-backed deterministic rules across six dimensions (frontmatter, description, tool hygiene, model fit, body structure, anti-patterns). Runs on all 14 sentra-hub + 42 orch agents without crashing. Every rule cites Anthropic spec or published prior art. (#10)
- **`/evaluate-agent --deep` — LLM-as-judge (Phase 2, skeleton)** — per-dimension isolated calls, evidence-before-score, file-hash + model + rubric-version cache. Judge model swappable via `ANTHROPIC_JUDGE_MODEL` (default: `claude-sonnet-4-6`). Falls back to dry-run when the SDK is missing. (#11)
- **`/evaluate-agent-behavior` skill — behavioral harness (Phase 3, skeleton)** — replay grader scores tool-whitelist adherence and step efficiency from a transcript. Fires `behavioral.tool_whitelist_violation` (critical) on a domain-engineer transcript that calls undeclared tools. (#12)
- **Output schema v1 + Markdown renderer** — single shape for all three evaluator phases. JSON Schema at `lib/eval/schema/v1.json`, jq-based renderer at `lib/eval/render.sh`. Exit codes 0/1/2 for ship/revise/reject. (#9)
- **8 agent + 4 skill archetypes** — orchestrator, tech-leader, domain-engineer, designer, qa-engineer, security-engineer, devops-engineer, monitor; domain-skill, coordination-skill, lint-skill, init-skill. Frontmatter follows orch's superset (`name, description, model, color, memory, maxTurns, tools, background?, effort?, isolation?`). 4 filled-in fixtures. (#3)
- **13 numbered rule templates** — plan-first, service-boundaries, definition-of-done, clarify-unknowns, interrupt-handling, small-wins, parallel-execution, client-first-communication, no-quick-fixes, style-per-service, no-push-without-permission, security-vapt, model-selection. Each opens with a "When this applies" header so `/team-gen` can pick a subset. (#5)
- **4 hook templates** — notify (cross-platform), session-init, teammate-idle-gate, post-commit-check. Plus a `settings.json.partial` the scaffolder merges into the generated team's settings. (#6)
- **Plugin scaffolding** — `plugin.json`, layout, plugin `CLAUDE.md`, root README, MIT LICENSE. (#2)
- **Documentation** — root README + `docs/{team-gen,meet,evaluator,contributing}.md` with cross-links and runnable examples. (#8)
